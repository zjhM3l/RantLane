# RantLane — 骂了么？

[English](README.md) | [中文说明](README_zh-CN.md)

> 一个略带邪性、尽量 CarPlay 友好的「车上吐槽 / 夸一嘴」语音实验项目。  
> 目标是让你**绕着别人说话**，而不是真的摇下车窗对喷。

RantLane 的灵感来自一个非常具体的瞬间：  
我们一起在奥斯汀自驾，高速上同行朋友来了一个*离谱的加塞*，后车直接闪灯 + 竖中指，全车笑疯。

回去路上大家开始脑洞：

> “要是车和车之间有个**按住就说的对讲机**就好了，  
> 还能自动翻译、算积分、有冷却时间，这样你既能吐槽、也能夸夸，  
> 又不用真的摇下车窗对线。”

这个仓库，就是想把这份灵感从「车上嘴炮」变成一个真实的 App Demo。

---

## 这是什么？

RantLane 是一个**实验性质、非生产环境**的项目，目标是：

- 探索附近车辆之间的 **基于位置的低延迟语音聊天**
- 学习如何做一个 **支持 CarPlay 的通信类 App**
- 顺便玩玩「**路怒但尽量安全**」的游戏机制  
  （冷却时间、「抖 M 积分」、沙雕成就等等）

你可以把它想象成：

- 司机 / 乘客的「车载对讲机」
- 围绕你这辆车的一个小型匿名广播电台
- 一个可以说“这波变道有点东西”或者“刚才让道谢谢”的地方

---

## 核心想法（持续脑洞中）

下面这些都还不是最终设计，更像是一个 backlog + 点子池。

### 1. 附近车辆语音链接

- 手机定期上报一个**粗粒度位置** → 服务端把设备塞进若干本地 **“气泡房间”**
- 每个房间就是一个轻量级语音房：
  - 按住说话 / 短语音 或 低延迟语音流
  - 可以完全 **匿名**（只展示一个头像 / 小车表情）
- 你可以：
  - 加入默认的「我周围」频道
  - 自己开一个车队/convoy 频道，路上一车队一起聊天

### 2. 可选的「开放对讲」总开关

- 一个很大的总开关：`开放周边对讲 [开 / 关]`
- **关**：你对陌生人完全隐身，只和队友 / 好友说话
- **开**：你同意：
  - 接收来自附近用户的**短语音**（且有频率限制）
  - 系统强制冷却，不能一直刷

### 3. 语音 + 翻译

毕竟路上的人不一定说同一种语言：

- 收到语音 → 语音转文字 → 机器翻译 → 再转回语音播放
- 优先目标语言：**英语 / 西语 / 中文**，后面可以慢慢加
- 理论上你可以用中文说 “谢谢你刚才让道”，  
  然后乘客 / 对方用西语/英语听到翻译版

### 4. 「抖 M 积分」& 成就体系

这是整套的梗 & 游戏化层：

- **Dō-M points（抖 M 积分，暂定名）**：
  - 冷静接收反馈（无论夸/吐槽） → 加分
  - 情绪过激、被多人拉黑 → 扣分
- 一些可能的成就：
  - `Chased but Praised` – 追上去只是为了夸一嘴  
  - `Zen Master` – 连续收到 10 条阴阳怪气，从未回喷
  - `Too Salty to Drive` – 过于“咸”，被系统临时静音

整个体系的目标是：**鼓励解压 & 幽默，而不是鼓励真正的谩骂 / 骚扰**。

### 5. 「往前转发」链（Pass It Forward）

- 比如你后面的车给你发了条（礼貌版）信息：  
  *“你前面那位好像忘了打转向灯……”*  
- 你可以选择 **转发一个被「润色过」的版本**给前面的车，  
  尽量保留信息，但弱化火药味。

更像是一个「社交实验」，而不是硬需求。

## CarPlay & 安全性

中长期目标是做出一个**支持 CarPlay 的通信 App Demo**：

- 使用 Apple 提供的 **CarPlay + SiriKit / CallKit** 通信类模式
- UI 设计原则：
  - 大号模板组件（按钮少、字大）
  - Siri / 语音控制优先
  - 尽量降低驾驶员分心

考虑到安全和 App Store 审核规则，本项目会明确：

- **不会**鼓励危险驾驶或现实中的骚扰行为
- 会提供：
  - 拉黑 / 静音 / 举报等基础风控手段
  - 发送频率限制 & 冷却时间
  - 可选「乘客模式」（例如车速超过某值时自动弱化功能）
- 并在显著位置注明：

> 这是一个娱乐 & 实验项目。  
> 请不要以任何会分散你驾驶注意力的方式使用本应用。

它更像是一个学习 & 快速验证想法的 playground，  
而**不是**任何形式的正式安全系统或法律建议。

## 技术栈（初步设想）

后面可能会变，但大致方向是：

- **客户端**
  - iOS（Swift / SwiftUI）+ CarPlay 支持
  - 之后可能补个 Android 版本
- **后端**
  - 实时信令：WebSocket / gRPC
  - 音频：WebRTC 或类 PTT 的语音流服务
  - 存储：用户信息、积分、基础日志
- **语音 / 算法**
  - 云厂商现成的语音识别 & 翻译
  - 轻量级敏感词 / 有害内容过滤

欢迎 PR 提出更合理的架构方案和技术选型。
